\# Інструкція з генерації та перегляду документації API



У проєкті \*\*PersonalFinanceSystem\*\* (компонент API) документація генерується автоматично на основі нативних XML-коментарів C# (`///`). 

Для стандартизації та генерації специфікації використовується вбудований пакет \*\*Microsoft.AspNetCore.OpenApi\*\*.



\## Як переглянути документацію:

1\. Переконайтеся, що ви знаходитесь у кореневій папці проєкту.

2\. Виконайте збірку для генерації актуального XML-файлу: `dotnet build`

3\. Перейдіть у папку API: `cd src/PersonalFinanceSystem.API`

4\. Запустіть сервер: `dotnet run`

5\. Зверніть увагу на термінал — там буде вказано реальну адресу, на якій запустився сервер (наприклад, `http://localhost:5123`).

6\. Щоб отримати файл специфікації, додайте до цієї адреси шлях `/openapi/v1.json` (наприклад, `http://localhost:5123/openapi/v1.json`).



\## Стандарти для розробників:

\- Кожен новий ендпоінт (`MapGet`, `MapPost` тощо) повинен мати блок `<summary>`.

\- Для додаткового опису бізнес-логіки використовуйте тег `<remarks>`.

\- Моделі даних (Records, DTOs) повинні містити опис параметрів через `<param>`.
