\# Звіт з налаштування лінтингу (Лабораторна робота №4)



\## 1. Обраний інструмент

Для проєкту обрано \*\*dotnet format\*\* та конфігурацію \*\*.editorconfig\*\*. 

Це стандартний інструмент для .NET 10, який дозволяє підтримувати єдиний стиль коду (відступи, переноси, правила іменування).



\## 2. Процес налаштування

1\. Створено файл `.editorconfig` з базовими правилами Microsoft.

2\. Визначено правила: відступи — 4 пробіли, обов'язковий порожній рядок в кінці файлу, перенос фігурної дужки на новий рядок.



\## 3. Результати запуску та виправлення

\- \*\*Початкова перевірка:\*\* Команда `dotnet format whitespace --verify-no-changes` виявила помилки форматування у файлі `Program.cs`.

\- \*\*Автоматичне виправлення:\*\* Запущено команду `dotnet format`, яка автоматично привела код до стандарту.

\- \*\*Метрика успіху:\*\* Повторна перевірка не виявила жодних відхилень. Виправлено 100% знайдених проблем.



\## 4. Команди для запуску

\- Перевірка: `dotnet format whitespace --verify-no-changes`

\- Виправлення: `dotnet format`
