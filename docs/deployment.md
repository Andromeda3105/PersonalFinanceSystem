\# Інструкція з розгортання у Production (Deployment Guide)



\## 1. Вимоги до апаратного забезпечення

\* \*\*Архітектура:\*\* x64 або ARM64

\* \*\*CPU:\*\* Мінімум 2 ядра (рекомендовано 4 для стабільної роботи бази даних та API)

\* \*\*Оперативна пам'ять (RAM):\*\* Мінімум 4 ГБ

\* \*\*Диск:\*\* 20 ГБ SSD (швидкість диска критична для СУБД)



\## 2. Необхідне програмне забезпечення

\* \*\*ОС:\*\* Ubuntu 22.04 LTS

\* \*\*Середовище виконання:\*\* .NET 10.0 ASP.NET Core Runtime

\* \*\*СУБД:\*\* PostgreSQL 14 або новіше

\* \*\*Веб-сервер (Reverse Proxy):\*\* Nginx



\## 3. Налаштування мережі

\* Відкриті порти на Firewall (UFW):

&#x20; \* `80` (HTTP) та `443` (HTTPS) для доступу клієнтського застосунку.

&#x20; \* `22` (SSH) для віддаленого адміністрування.

\* Порт `5432` (PostgreSQL) повинен бути закритий для зовнішніх підключень і доступний лише локально (`localhost`).



\## 4. Конфігурація серверів та СУБД

1\. Встановіть PostgreSQL та створіть базу даних:

&#x20;  `CREATE DATABASE finance\_prod;`

2\. Налаштуйте Nginx для перенаправлення трафіку з порту 80 на локальний порт застосунку (5000).



\## 5. Розгортання коду

1\. Опублікуйте проєкт на машині збирання: 

&#x20;  `dotnet publish src/PersonalFinanceSystem.API -c Release -o /var/www/finance-api`

2\. Налаштуйте файл `appsettings.Production.json` з реальними паролями до БД.

3\. Створіть та запустіть systemd-сервіс для автоматичного старту бекенду.



\## 6. Перевірка працездатності

\* Перевірте статус служби: `sudo systemctl status finance-api`

\* Виконайте тестовий запит до сервера: `curl -I http://localhost:5000/health` (очікуваний результат: 200 OK).
